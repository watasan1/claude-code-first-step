# claude-code-first-step

本資料は、Claudeを初めて使う方が、アカウント作成からClaude Codeの導入、そして実際にHTML/CSS/JavaScriptでTodoリストアプリを完成させるまでを、一連の流れで体験できるようにまとめたものです。この手順に沿って進めることで、Claudeのセットアップからコード生成・修正・デバッグ・テスト・Gitでのバージョン管理まで、Claude Codeを使った基本的な開発フローをひと通り身につけることができます。

## 1. Claude desktopの導入

### Step 1 ダウンローダーをインストールする

公式サイトから、[https://claude.com/download](https://claude.com/download) 自分の端末にあったインストーラーボタンをクリックします。

### Step 2 Claude をアプリケーションフォルダにドラッグ&ドロップ

Claude desktop をダブルクリックすると。Claudeのアイコンを「Applications」フォルダのアイコンへドラッグ&ドロップしてインストールします。

### Step 3 アプリケーションを実行する

インストールしたClaudeアプリのアイコンをLaunchpadからダブルクリック（または1回クリック）して起動します。

macOS のセキュリティ確認ダイアログが表示されます。

開いてもよろしいでしょうか？ 開くボタンをクリックして起動を許可します。

### Step 4 「始める」　ボタンをクリックします

ウェルカム画面が表示されたら「始める」をクリックして設定を進めます。

## 3. Claude Codeの導入

[https://code.claude.com/docs/en/overview](https://code.claude.com/docs/en/overview) に従ってインストールする。

macOS:

```Bash
curl -fsSL https://claude.ai/install.sh | bash
```

✔ Claude Code successfully installed! というテキストを確認して完了です。

PATHを通す（zshの場合。bashの場合は ~/.bashrc に読み替え）:

```Bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc
```

## 3. Claude の操作に慣れる

### Step 1 作業フォルダの準備

1. Claude Code Desktop を実行すると「フォルダを選択」をクリックする

2. 「フォルダを開く」で claude フォルダを作成して選択する

例として claude フォルダを使用します。フォルダ名は任意です。

3. 「ワークスペースを信頼しますか？」画面が表示されたら「ワークスペースを信頼する」ボタンをクリックする

4. VSCode でプロジェクトを開く：/Users/アカウント/claude

### Step 2 Claude Code Desktopの構成画面

Chat ペイン（チャット入力欄＋会話エリア）

利用できる主なペイン：

Chat — チャット入力欄と会話エリアはこれ1つにまとまっています（別々の区画ではありません）
Diff — コード変更のレビュー
Browser — アプリのプレビューや外部サイト表示
Terminal — ターミナル操作
File — ファイルの閲覧・編集
Plan — 計画立案用ワークスペース
Tasks — バックグラウンド作業の監視
Subagent — サブエージェントの出力

ここでいう、ペインとは、「区画」や「パネル」で、Claude Code Desktopを区画のように、分割表示して開発していきます。

これらのペインはドラッグで配置変更、境界をドラッグしてリサイズが可能で、「Views」メニューから開閉できます（フォーカス中のペインを閉じるショートカットは Mac が Cmd+\、Windows が Ctrl+\）。

つまり「チャット入力欄」と「会話エリア」は別区画ではなく1つの Chat ペインにまとまっており、それに加えて Diff・Browser・Terminal・File など複数のペインを自分好みに配置できる、という構成です。

### Step 3 ファイル操作とコード生成

- Hello Worldを表示するJavaScriptスクリプトを作成してください
- hello.js をNode.jsで実行させてください
- index.htmlを作成して。シンプルなランディングページにして

### Step 4 変更のレビューと承認：

- 関数にエラーハンドリングを追加してください
- このファイル全体をTypeScriptに書き直してください

### Step 5 コードの品質をあげる：

- このコードをリファクタリングして、可読性を高めてください
- ESLint/Prettierのルールに準拠するようにコードを整形して
- コメントと型ヒント（JSDoc）を追加してください
- モデルとビューを分離するようにコードをリファクタリングして

### Step 6 コードの説明・デバッグ・テストを試す

- @script.js このコードが何をしているか、初心者向けに説明してください （@メンションでファイルを指定する）
- この正規表現の意味を解説して
- このアルゴリズムのを教えて
- このエラーを修正してください

### Step 7 Git連携を試す

Gitの操作をAIを通じて行えます。ターミナルでgitコマンドを打つ必要はありません。

- 今の変更をコミットしてください
- わかりやすいコミットメッセージを考えて、コミットして
- 変更内容を確認して、コミットメッセージを日本語で提案して

CLAUDE.md：AIへの永続的な指示書

プロジェクトフォルダに CLAUDE.md ファイルを置くと、Claude Codeが起動するたびにその内容を読み込みます。チームのルールや注意事項を書いておくと便利です。
